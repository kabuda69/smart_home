# 智能家居模拟控制系统技术文档

## 1. 项目概述

智能家居模拟控制系统是一个基于 Spring Boot + Vue 的全栈应用，旨在提供完整的智能家居设备管理、监控和控制功能。系统支持设备状态实时监控、阈值警报、场景模式、数据统计分析等核心功能，为用户提供便捷的智能家居管理体验。

### 1.1 系统目标

- **实时监控**：通过WebSocket实现设备状态实时更新和警报通知
- **智能控制**：支持设备开关、数值调节和场景模式一键控制
- **数据分析**：提供设备使用统计和警报趋势分析
- **安全可靠**：基于JWT的认证系统和完善的权限控制
- **用户友好**：现代化的前端界面和流畅的用户体验

### 3.2 核心流程

1. **用户认证流程**
   - 用户登录 → 验证 credentials → 生成 JWT Token → 前端存储 Token
   - 后续请求携带 Token → 后端验证 Token → 授权访问

2. **设备控制流程**
   - 前端发送控制命令 → 后端接收并验证 → 执行设备操作 → 更新设备状态 → WebSocket推送状态更新

3. **警报处理流程**
   - 设备状态变化 → 检查阈值 → 触发警报 → 存储警报记录 → WebSocket推送警报通知

4. **场景激活流程**
   - 前端触发场景激活 → 后端执行场景动作 → 批量控制设备 → 更新场景状态

## 4. 功能模块

### 4.1 用户认证模块

- **注册/登录**：支持新用户注册和现有用户登录
- **JWT认证**：基于Token的无状态认证机制
- **权限控制**：区分普通用户和管理员权限
- **会话管理**：前端本地存储Token和用户信息

### 4.2 设备管理模块

- **设备绑定**：支持添加新设备并选择设备类型
- **状态监控**：实时显示设备在线状态和当前数值
- **设备控制**：支持设备开关和数值调节
- **阈值设置**：可为设备设置最小/最大阈值
- **历史记录**：查看设备状态历史变化

### 4.3 实时监控模块

- **WebSocket推送**：设备状态变化实时更新
- **动态告警弹窗**：阈值超限时触发居中弹窗，带动画效果和提示音
- **警报通知**：及时推送设备异常信息
- **状态同步**：多设备状态实时同步

### 4.4 场景模式模块

- **场景创建**：支持创建自定义场景（如回家模式、离家模式）
- **动作配置**：可为场景添加多个设备动作（开启/关闭/设置数值）
- **一键激活**：单个按钮触发多个设备操作
- **场景管理**：支持场景编辑和删除

### 4.5 数据统计模块

- **设备统计**：设备总数和在线设备数量统计
- **类型分布**：设备类型占比饼图
- **警报趋势**：每月警报数量柱状图
- **命令频率**：设备命令执行频率分析
- **实时数据**：使用真实历史数据进行统计

### 4.6 设备分享模块

- **临时分享**：生成24小时有效的分享链接
- **快照查看**：无需登录即可查看设备状态
- **自动清理**：过期分享自动失效

### 4.7 系统设置模块

- **通知偏好**：控制弹窗通知和日志记录
- **操作日志**：查看个人操作历史
- **时间查询**：支持按时间段查询操作日志

### 4.8 管理后台模块

- **用户管理**：查看和管理系统用户
- **设备类型**：管理系统支持的设备类型
- **反馈管理**：查看和回复用户反馈
- **系统日志**：查看系统级操作日志

## 7. 部署指南

### 7.1 环境要求

- JDK 11+ 
- MySQL 8.0+ 
- Node.js 14+ 
- npm 6+ 
- Maven 3.8+

### 7.2 数据库配置

1. 创建MySQL数据库：
   ```sql
   CREATE DATABASE smart_home CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
   ```

2. 修改 `src/main/resources/application.properties`：
   ```properties
   spring.datasource.url=jdbc:mysql://localhost:3306/smart_home
   spring.datasource.username=root
   spring.datasource.password=你的密码
   ```

### 7.3 后端启动

```bash
# 使用Maven
mvn spring-boot:run

# 或使用Maven Wrapper
./mvnw spring-boot:run
```

后端启动后会自动创建表结构和初始数据。

### 7.4 前端启动

```bash
cd frontend
npm install
npm run serve
```

### 7.5 访问系统

- 前端：http://localhost:8081
- 后端API：http://localhost:8080
- 默认管理员：admin / admin123
- 默认用户：user1 / 123456

## 11. 故障排查

### 11.1 常见问题

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 登录失败 | 密码错误或用户不存在 | 检查用户名密码，确认用户状态 |
| 设备控制无响应 | 网络连接问题 | 检查网络连接，重启设备 |
| 警报不触发 | 阈值设置错误 | 检查设备阈值配置 |
| 场景激活失败 | 设备离线 | 检查设备在线状态 |
| WebSocket连接失败 | 服务器端口被占用 | 检查8080端口是否被占用 |
